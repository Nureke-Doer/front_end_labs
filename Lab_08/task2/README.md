## Performance Comparison: SSR vs SSG

In this lab, we compared Server-Side Rendering (SSR) and Static Site Generation (SSG) by creating two identical pages (`/about-ssr` and `/about`). 

After running the production build (`npm run build` and `npm run start`), the differences became clear:

1. **SSG (`/about`):** The timestamp generated by `getStaticProps` remained constant across multiple page refreshes. The page loaded instantly because the HTML was already pre-rendered during the build time.
2. **SSR (`/about-ssr`):** The timestamp generated by `getServerSideProps` updated on every single page refresh. This proves that the server is actively processing the request and generating fresh HTML each time a user visits the page.

**Conclusion:** SSG is much faster for the initial load and reduces server stress, making it ideal for static content. SSR guarantees absolute data freshness but requires the server to work on every request, making it necessary for personalized views like the User Dashboard.
